你对浏览器内核的理解？主流浏览器内核？

解析：
①浏览器内核包括两部分，渲染引擎和js引擎。渲染引擎负责读取网页内容，整理讯息，计算网页的显示方式并显示页面，js引擎是解析执行js获取网页的动态效果。 后来 JS 引擎越来越独立，内核就倾向于只指渲染引擎。

②分类如下：
1、IE浏览器内核：Trident内核，也是俗称的IE内核；
2、Chrome浏览器内核：Blink内核；
3、Firefox浏览器内核：Gecko内核，俗称Firefox内核；
4、Safari浏览器内核：Webkit内核；
5、Opera浏览器内核：Blink内核；
6、360浏览器、猎豹浏览器内核：IE+Chrome双内核；
7、搜狗、遨游、QQ浏览器内核：Trident（兼容模式）+Webkit（高速模式）；
8、百度浏览器、世界之窗内核：IE内核；
9、2345浏览器内核：IE+Chrome双内核；
10、UC浏览器内核：U3内核（基于Webkit）

页面导入样式时，使用link和@import有什么区别

答：①link是HTML标签，除了加载CSS外，还能通过修改属性实现其他用法；@import是CSS提供的，只能用于加载CSS
②link引用CSS时，在页面载入时同时加载；@import需要页面网络完全载入以后加载
③link是HTML标签，不存在兼容性问题；@importCSS2.1提出，低版本（IE5以下）的浏览器不支持
④link支持javascript控制DOM去改变样式；@import不支持

标准CSS盒子模型？IE盒子模型？box-sizing属性？

解析：

  1. 标准CSS盒子模型
    实际所占宽度=内容的宽度（content） + padding+ border + margin
  2. IE盒子模型
    实际所占宽度=内容宽度（content+border+padding）+ margin
  3. box-sizing
    ①content-box //w3c盒模型 默认值，实际占地宽高=width+内边距+边框+外边距

    ②border-box //IE盒模型 给定的width中，会包含padding以及border,也就是说实际占地宽高=width+外边距

常见的兼容性问题

  1. 不同浏览器的标签默认padding和margin不一样。通常使用*{margin:0;padding:0;}
  2. IE6双边距bug：块属性标签float后，又有横行的margin情况下，在IE6显示margin比设置的大。hack：display:inline;将其转化为行内属性。
  3. 渐进识别的方式
    {
    background-color:#f1ee18;/*所有识别*/
    .background-color:#00deff\9; /*IE6、7、8识别*/
    +background-color:#a200ff;/*IE6、7识别*/
    _background-color:#1e0bd1;/*IE6识别*/
    }
  4. IE6,IE7中高度超出自己设置的高度。hack：①给超出高度的标签设置overflow:hidden②设置行高line-height小于你设置的高度
  5. IE下，可以使用获取常规属性的方法来获取自定义属性，也可以使用getAttribute()获取自定义属性；Firefox下，只能使用getAttribute()获取自定义属性。解决方法：同意通过getAttribute()获取自定义属性。
  6. Chrome中文界面下默认会将小于12px的文本强制按照12px显示，可通过加入CSS属性-webkit-text-size-adjust:none;解决。
  7. 超链接访问过后hover样式就不出现了，被点击访问过的超链接样式不再具有hover和active了。改变CSS属性的排列顺序：L-V-H-A(love hate):a:link{} a:visited{} a:hover{} a:active{}

为什么使用初始化CSS样式

由于浏览器的兼容性问题，不同浏览器对标签的默认值是不同的，如果没有做初始化不同浏览器就会出现页面显示差异。

css3动画

答： 哦，您问的css3动画啊，css3动画大致上包括两种：
第一种：过渡动画：主要通过transition来实现，通过设置过渡属性，运动时间，延迟时间和运动速度实现。
第二种：关键帧动画：主要通过animation配合@keyframes实现
transition动画和animation动画的主要区别有两点：
第一点transition动画需要事件来触发，animation不需要
第二点:transition只要开始结束两种状态，而animation可以实现多种状态，并且animation是可以做循环次数甚至是无限运动

em和em的区别

答： rem和em都是相对单位，主要参考的标签不同：
rem是相对于根字号，即相对于<html>标签的font-size实现的，浏览器默认字号是font-size:16px
em:是相对于父元素标签的字号，和百分比%类似，%也是相对于父级的，只不过是%相对于父级宽度的，而em相对于父级字号的

手机端如何做适配的

答： 前端做适配没有最好的方法，只有适合的方法，目前前端主要做适配的方法有：百分比，em,rem,媒体查询(即media query),flex布局（即弹性盒），vw,vh等
目前我在项目中用的多的是rem，flex布局，有时会用到媒体查询，在做pc响应式布局时用
主要是用了一个手淘的js库flexible.js,在页面变化时,检测页面宽度,除以10份,动态的赋值给font-size.属性.;而页面的布局我是通过rem来进行布局的,所以就可以适配所有的移动端设备

vw和vh了解吗

答： vw和vh是最近2年才慢慢火起来的css布局单位，现在已经被一些公司在使用，vw和vh分别相对于屏幕宽度和屏幕高度的，1vw相当于屏幕宽度的1%,100vw相当于满屏宽度100%,vh和vh类似，只不过是相对于屏幕高度的，1vh相当于屏幕高度的1%,100vh相当于满屏高度的100%

4. 谷歌浏览器如何显示12px以下的字号
中文版的chrome有个12px字体限制的问题，就是当字体小于12px时候都以12px来显示，这个问题在中文网站中并不突出，因为中文字体为了显示清晰一般都定义为大于或等于12px，但如果是一些英文网站那就不好说了，这时12px的限制就会破坏页面的美感，甚至因为文字变大而导致页面变形。
以前有个属性#chrome10px{ -webkit-text-size-adjust:none; font-size:10px; },但是新版谷歌已经不起作用了.我们可以通过css3的缩放来实现这个问题,比方说我要展示10px的文字,我可以通过设置字体20px,然后scale(0.5);

一个网页从输入url到呈现页面的过程？

简单介绍：
①域名解析
当输入www.xxx.com时，互联网会对输入的网址进行域名解析，解析为IP地址(互联网上每台机器的唯一标识是IP地址)。
②TCP链接
HTTP协议是使用TCP协议(传输控制协议)作为其传输层协议的，在拿到IP地址时，浏览器客户端会与服务器端建立TCP连接。
③浏览器发送HTTP请求
浏览器通过TCP协议将请求报文传送到服务器的指定接口。报文分为请求行(指定的请求方法、url、http协议版本)、请求头(浏览器的相关信息、语言、编码)、请求正文(当发送请求时，储存来自向服务器传递的数据)。
④服务器处理请求
返回响应报文(状态码、响应头、响应正文)。

  • 状态码

    状态码 含义
    1** 服务器接收成功
    2** 请求成功
    3** 重定向，请求的响应在另一个URL上找到
    4** 客户端错误
    5** 服务器错误

  • 响应头
    包含响应的相关信息

  • 响应正文
    服务器返回给浏览器的文本信息(HTML、JS、Css、图片等部分信息)

⑤浏览器页面渲染
浏览器接收到HTTP服务器传来的响应报文，并解析HTML文档，解析时实现渲染页面(重点：解析HTML来构建DOM树，构建渲染树，布局页面元素，绘制页面元素)。
⑥断开TCP连接
客户端发送请求给服务器，服务器接收到请求后做好准备，通知客户端，客户端发送消息给服务器确定断开，服务器断开连接。

纯css实现tab切换效果

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>css实现TAB</title>
  <style>
    /* 清除所有内外边距 */
    * {
      margin: 0;
      padding: 0;
    }

    /* 菜单 */
    .menu {
      width: 300px;
      height: 100px;
      border: solid 2px #e65151;
    }
    /* 主题定位，定义大小 */
    .list {
      position: relative;
      width: 300px;
      height: 100px;
    }
    /* 去除列表样式，导航栏浮动 */
    .list-item {
      list-style: none;
      float: left;
      width: 100px;
      height: 20px;
    }

    /* 单选框隐藏 */
    .list-item .tb-radio {
      position: absolute;
      display: block;
      width: 100px;
      height: 20px;
      opacity: 0;
    }


    /* 导航栏 */
    .list-item .tb-label {
      display: block;
      width: 100px;
      text-align: center;
      line-height: 20px;
      background: peru;
    }

    /* 相对定位覆盖 */

让一个元素居中显示

解析：

<div class="parent">
  <div class="child"></div>
</div>
<style>
.parent {
    width: 300px;
    height: 300px;
    background-color: blue;
}
.child {
    width: 100px;
    height: 100px;
    background-color: red;
}
</style>

① absolute + 负 margin 

.parent {
    position: relative;
}
.child {
    position: absolute;
    top: 50%;
    left: 50%;
    margin-top: -50px;
    margin-left: -50px;
}

② absolute + transform 

.parent {
    position: relative;
}
.child {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

③ absolute + margin auto 

.parent {
    position: relative;
}
.child {
    position: absolute;
    top: 0;
    left: 0;
    right: 0;
    bottom: 0;
    margin: auto;
}

④ absolute + calc 

.parent {
    text-align: center;
    line-height: 300px; /* 等于 parent 的 height */
}
.child {
   display: inline-block;
   vertical-align: middle;
   line-height: initial; /* 这样 child 内的文字就不会超出跑到下面 */
}

⑤ table-cell 

.parent {
    display: table-cell;
    text-align: center;
    vertical-align: middle;
}
.child {
   display: inline-block;
}

⑥ Grid 

.parent {
    display: grid;
}
.child {
   align-self: center;
   justify-self: center;
}

⑦ writing-mode

.parent {
    writing-mode: vertical-lr;
    text-align: center;
}
.child {
   writing-mode: horizontal-tb;
   display: inline-block;
   margin: 0 calc(50% - 50px);
}
