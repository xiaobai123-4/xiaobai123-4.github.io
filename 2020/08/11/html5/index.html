  1. HTML 侧边分节元素 (``) 定义了这样的节段, 虽然是主要的分节元素, 但并不属于主要的文档流, 就像解释栏或广告栏. aside元素内部有自己的大纲，但并不计入文档大纲中

WebSocket–网络协议
WebSocket网络协议:由于它的工作方式特殊(广播与收听),所以在移动项目中大量使用
HTTP 网络协议(超文本传输协议)–(协议程序)
工作方式:请求头;响应头;响应状态码
HTTP作用:传输网页中资源(html;css;js;img;avi;mp3…)
HTTP:网页搬运工(从服务器将资源运输浏览器)
HTTP工作方式:请求和响应
请求消息:客户端浏览器发送服务器数据称:请求消息
响应消息:服务器发送数据给浏览器称:响应消息
一次请求对应一次响应,如果没有请求就没有响应

注意:有一些应用中使用HTTP协议不适应

实例：

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    canvas {
      background-color: aquamarine;
    }
  </style>
</head>

<body>
  <h2>canvas绘图</h2>
  <!-- 矩形 -->
  <canvas id="jx" width="400" height="400"></canvas>
  <!-- 圆形 -->
  <canvas id="yx" width="400" height="400"></canvas>
  <!-- 三角形 -->
  <canvas id="sjx" width="400" height="400"></canvas>
  <h2>canvas渐变</h2>
  <!-- 渐变 -->
  <canvas id="jb" width="400" height="400"></canvas>
  <!-- 放射性渐变 -->
  <canvas id="fsxjb" width="400" height="400"></canvas>
</body>
<script>
  // 矩形
  // 获取canvas区域
  var canvas1 = document.getElementById("jx");
  // 获取绘图canvas的上下文
  var ctx1 = canvas1.getContext("2d");
  // 指定线条颜色
  ctx1.strokeStyle = "#f34851";
  // 指定内部区域颜色
  ctx1.fillStyle = "#f34851";
  // 绘制一个带描边的矩形(x,y,width,height)
  ctx1.strokeRect(10, 10, 260, 260);
  // 绘制一个实心的矩形
  ctx1.fillRect(20, 20, 240, 240);
  // 清除一个矩形形状的区域
  ctx1.clearRect(30, 30, 220, 220);

  // 圆形
  // 获取canvas区域
  var canvas2 = document.getElementById("yx");
  // 获取绘图canvas的上下文
  var ctx2 = canvas2.getContext("2d");
  // 开始圆形绘制
  ctx2.beginPath();
  // 圆形参数
  ctx2.arc(200, 200, 100, 0, Math.PI * 2, false);
  // 指定内部区域颜色
  ctx2.fillStyle = "red";
  ctx2.fill();
  ctx2.closePath();

  
  
  // 三角形
  // 获取canvas区域
  var canvas = document.getElementById("sjx");
  // 获取绘图canvas的上下文
  var ctx3 = canvas.getContext("2d");
  // 指定线条颜色
  ctx3.strokeStyle = "#f34851";
  // 开始绘制记录的原点位置
  ctx3.beginPath();
  // 移动原点位置到(100,100)
  ctx3.moveTo(100, 200);
  // 在当前位置开始绘制一条直线到(200,100)
  ctx3.lineTo(200, 200);
  // 在绘制一条直线到(200,0)
  ctx3.lineTo(100, 10);
  // 关闭
  ctx3.closePath();
  ctx3.stroke();


  // 渐变
  // 获取canvas区域
  var canvas = document.getElementById("jb");
  // 获取绘图canvas的上下文
  var ctx4 = canvas.getContext("2d");
  var grad = ctx4.createLinearGradient(0, 0, 400, 400);
  // 绘制一个带描边的矩形(x,y,width,height)
  grad.addColorStop(0, "red");
  grad.addColorStop(0.5, "yellow");
  grad.addColorStop(1, "blue");
  // 设置区域颜色值
  ctx4.fillStyle = grad;
  ctx4.fillRect(20, 20, 240, 240);
</script>

通信

Web Sockets：允许在页面和服务器之间建立持久连接并通过这种方法来交换非 HTML 数据。

<!DOCTYPE HTML>
<html>
<head>
<script type="text/javascript">
  function WebSocketTest() {
    if ("WebSocket" in window) {
      alert("您的浏览器支持 WebSocket!");
      // 打开一个 web socket
      var ws = new WebSocket("ws://localhost:5500");
      ws.onopen = function () {
        // Web Socket 已连接上，使用 send() 方法发送数据
        ws.send("发送数据");
        alert("数据发送中...");
      };
      ws.onmessage = function (evt) {
        var received_msg = evt.data;
        alert("数据已接收...");
      };
      ws.onclose = function () {
        // 关闭 websocket
        alert("连接已关闭...");
      };
    } else {
      // 浏览器不支持 WebSocket
      alert("您的浏览器不支持 WebSocket!");
    }
  }
</script>
</head>

<body>
  <div id="sse">
    <a href="javascript:WebSocketTest()">运行 WebSocket</a>
  </div>
</body>
</html>

性能

webworker
Web Workers API：一个worker是使用一个构造函数创建的一个对象(e.g. Worker()) 运行一个命名的JavaScript文件 - 这个文件包含将在工作线程中运行的代码; workers 运行在另一个全局上下文中,不同于当前的window. 因此，使用 window快捷方式获取当前全局的范围 (而不是self) 在一个 Worker 内将返回错误。
作用:新线程用于执行js中比较耗时任务
程序:在磁盘中保存代码称为程序 01.html 01.js
进程:将磁盘中代码调用内存中进行，在内存中代码为”进程”
线程:进程内部由于多个线程组成
#报错
#浏览器强型规定:DOM/BOM 只能UI执行其它线程不可以
(1)创建Worker线程 var w = new Worker(“01.js”)
(2)接收数据 w.onmessage=function(e){e.data}
(3)发送数据 w.postMessage(str)
小结
-WebWorker：是一个新线程用于执行耗时js任务
-要求:不能执行任务DOM/BOM代码，jquery不行
-三条语句

  • var w = new Worker(“js程序路径名”) #创建新线程
  • w.postMessage(str) #发送数据
  • w.onmessage = function(e){e.data} #接收数据
