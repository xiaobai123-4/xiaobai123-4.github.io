拷贝

2020-07-18 javascript 787 字
前端 javascript 知识点

引入

变量包含两种不同数据类型的值：基本数据类型和引用数据类型。
基本数据类型：名值存储在栈内存中；
引用数据类型：名存在栈内存中，值存在于堆内存中，但是栈内存会提供一个引用的地址指向堆内存中的值。

目前基本数据类型有：Boolean、Null、Undefined、Number、String、Symbol，引用数据类型有：Object、Array、Function、RegExp、Date等

深拷贝与浅拷贝的概念只存在于引用数据类型

拷贝分为浅拷贝和深拷贝，我们可以理解拷贝是一种复制行为，例如

//①变量a的值拷贝给了变量b，从b也变为了1
let a=1;
let b=a;
b=3;
//变量b赋值为3，a还是1，b是3，也就是说a和b是相互对立的，相互影响的，这就是按值传递
//②变量d拥有和变量c同一个对象
let c={};
let d=c;
//对象，数组这些是引用传递
let left={a:1};
let right=left;
left.a=5;
//right.a也变成了5，也就是说a和b会相互影响，指向同一个对象，这就是引用类型

按值传递：在两变量间赋值时，或将变量作为参数传入函数时，仅将原变量中的值复制一个副本给对方，传递后两个变量各不相干

浅拷贝和深拷贝

浅拷贝

只复制一层对象，当对象的属性是引用类型时，实质上复制的是其引用，当引用指向发生变化的时候，原对象属性值也跟着变化，相互还是会受到影响。

深拷贝

在拷贝的时候，创建新的对象，并把原对象所有属性都深拷贝进新对象，原属性如果是对象，也会重新创建新的对象并拷贝到新对象属性中，这样新旧对象，是相互对里的，互不影响，这个就是深拷贝

var lilei1 = {
  name: "李磊",
  age: 12,
  address: {
    city: "西安",
    street: "体育场"
  }
}
// 浅拷贝
function clone(oldObj) {
  if (oldObj == null) {
    return null
  }
  if (typeof (oldObj) !== "object") {
    return oldObj
  }
  var newObj={}
  var newObj = Array.isArray(oldObj) ? [] : {};
  for (var key in oldObj) {
    newObj[key] = oldObj[key]
  }
  return newObj;
}

var lilei2 = clone(lilei1)
lilei2.age=33
lilei2.address.city="北京"
lilei2.address.street="五棵松"
console.log(lilei1)
console.log(lilei2)

// 特定方法Object.assign({}, obj)
lilei5=Object.assign({}, lilei1)
lilei5.age=22
lilei5.address.city="汉中"
lilei5.address.street="郑国渠"
console.log(lilei1)
console.log(lilei5)

// 深拷贝
function deepclone(obj) {
  if(obj==null){return null}
  if(typeof(obj)!=="object"){return obj}
  var newobj = Array.isArray(obj)?[]:{};
  for (var key in obj) {
    newobj[key]=deepclone(obj[key])
  }
  return newobj
}
var lilei3=deepclone(lilei1)
lilei3.age=13
lilei3.address.city="成都"
lilei3.address.street="酒馆"
console.log(lilei1)
console.log(lilei3)//13,成都，酒馆

// JSON.parse(JSON.stringify( obj ))
lilei6=JSON.parse(JSON.stringify(lilei1))
lilei6.age=13
lilei6.address.city="成都"
lilei6.address.street="酒馆"
console.log(lilei1)
console.log(lilei6)

深拷贝就是完全解除引用关系，产生新的对象，与原来的互不影响
