利用Hexo搭建个人博客

安装前提

  • Node.js (Node.js 版本需不低于 8.10，建议使用 Node.js 10.0 及以上版本)
  • Git
  • 注册GitHub账号

安装Hexo

利用npm安装Hexo

//国内使用淘宝镜像
$ npm install -g cnpm --registry=https://registry.npm.taobao.org
//安装hexo脚手架
$ npm install -g hexo-cli
  • 验证hexo安装是否成功hexo -v

创建本地博客站点

## 创建项目
$ hexo init myBlog
## cd到项目文件夹
$ cd myBlog
## 安装相应包
$ npm install

完成上述操作以后，指定文件夹的目录下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

测试本地博客站点

## 生成文章
$ hexo new "我的第一篇博客"
## 部署之前生成静态文件
$ hexo g
## 部署文件
$ hexo d
## 启动服务
$ hexo s

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

$ hexo clean

上传到GitHub

新建一个仓库Create a new repository

名字最好是username+github.io

获得仓库地址

修改_config.yml

deploy:
  type: git
	repository: https://github.com/username/username.github.io.git
	branch: master

git中

$ npm install hexo-deployer-git -save
$ hexo g
$ hexo d

相关配置

  1. 配置个人信息

    # Site
    title: 小白的博客
    subtitle: 做自己的梦，走自己的路。
    description: 学习永不止步
    keywords:
    author: MaQisong
    language: zh-Hans
    timezone: ''

    next主题相关配置

    menu:
      home: /|| home
      about: /about/|| user
      tags: /tags/|| tags
      categories: /categories/|| th
      archives: /archives/|| archive
      schedule: /schedule/|| calendar

    1. 创建一个标签

      git命令

      hexo new page "tags"

      在生成的tags中的index.md中配置如下

      title: tags #汉语的话要改成标签
      date: 2020-05-23 13:30:51
      type: "tags" #增加type属性

      为文章添加标签：

      ---
      title: 测试
      tags: ["hello","我是标签"]
      ---

    2. 创建一个分类

      git命令

      hexo new page "categories"

      在生成的tags中的index.md中配置如下

      ---
      title: categories #分类
      date: 2020-05-23 13:30:51
      type: "categories" #增加type属性
      ---

      为文章添加标签：

      ---
      title: 测试
      tags: ["hello","我是标签"]
      ---

常见错误

  1. cannot get %20 
menu:
  home: /|| home
  about: /about/|| user
  tags: /tags/|| tags
  categories: /categories/|| th
  archives: /archives/|| archive
  #schedule: /schedule/ || calendar #官方给的代码 || 前多加了空格

  1. cannot GET/tags or categories

    原因：尚未初始化

    • 在git bash 中输入以下代码创建相应的page：hexo new page “tags”hexo new page “categories”

    • 在第一步完成后会在source文件夹中出现tags的文件夹，在各自的文件夹里打开index.md文件进行修改(多加上一个type属性)：

      title: tags
      date: 2020-05-15 14:20:32
      type: "tags" #增加type属性

  2. 日志点击后跳转乱码 Cannot GET /archives/%7C%7C%20archive/

    主题目录下/layout/_macro/sidebar.swig文件中找到<a href="{{ url_for(theme.menu.archives).split('||')[0] | trim }}">原因是url_for函数将||转码了，改成<a href="{{url_for(theme.menu.archives.split('||')[0])| trim}}">就可以了
