如何理解HTML语义化，为什么要使用语义化标签？

解析：
①HTML语义化：就是使用正确的标签来表达含义，常见的比如标题用h1-h6标签；段落用p标签等等
②为什么：让页面具有良好的结构和含义，可以有效提高以下几点：
a. 可访问性：无障碍阅读，方便转译
b. 可检索性：具有良好的结构和语义，可以提高搜索引擎的有效爬取，提高网站流量
c. 国际性：语义化标签方便更多开发者弄懂你的网页结构
d. 互用性：减少网页间的差异性，帮助其他开发者了解你网页的结构，方便后期开发和维护

Css

一、什么是css

英文名：Cascading style sheets
中文名：层叠样式表、级联样式表、简称样式表
作用：修饰和美化html网页，css样式与html属性相冲突的话，w3c的建议是尽量使用css样式来取代html属性
*html:网页的结构的搭建，数据的显示 | css:构建html网页的样式，装饰页面

二、css语法

  1. css样式表
    内联样式：将样式定义在单个html元素中
    语法：<any style="样式声明1;样式声明2;...">
    注意：
    ①必须在开始标记中声明style属性，里面是若干样式声明
    ②样式声明 修饰当前元素的效果是什么

    样式声明：修饰当前的元素的效果是什么。样式属性:样式值/属性值;...
    例子:<any style="color:red;font-size:24px">

    常用样式属性和值
    ①颜色
    属性：color
    属性值：颜色值
    ②背景色
    属性：background-color（同color）
    ③字体大小
    属性：font-size
    属性值：以px为单位的数值
    特点：
    ①内联样式不能复用
    ②内联样式优先级最高
    项目中，一般极少使用内联样式，通常在学习和测试的时候使用

  2. 内部样式表
    定义：将样式的内容放在网页的<head></head>标签定义的style标记中
    语法：

    <head>
      <style>
        样式规则1
        样式规则2
        样式规则3
      </style>
    </head>

    样式规则的结构：
    选择器：规范了页面中那些元素能够使用声明好的样式
    例如：div{} p{} h2{}

    语法：

    选择器{
      样式声明1；
    }

    特点：
    ①样式可以重用，但是仅对当前页面元素生效
    ②内部样式的重用有局限性，项目中使用的较少，在学习测试中使用较多

  3. 外部样式表
    定义：将样式规则定义在独立的样式表文件中(xx.css)，在使用网页时，可以将样式表文件进行引入
    步骤：
    ①声明.css文件 并编写样式
    ②在网页中对css文件进行引用

    <head>
     <link rel="stylesheet" href="**.css">
    </head>

    特点：可以样式重用

三、css样式表的特征

继承性
父元素的一些样式，可以给子元素(子元素不用定义直接拥有)
层叠型
可以为一个元素设置多个样式规则，多个样式规则的样式声明不冲突时，那么都可以应用在当前元素上
在层叠型的基础上，如果样式声明发生冲突的话，则按照不同使用方式的优先级来应用样式
优先级
优先级低：浏览器的缺省设置(默认) user agent styleSheet
优先级中：内部样式表和外部样式表
当样式表发生冲突用就近原则，后定义者优先
优先级高：内联方式
优先级极高：!important
调整优先级
!important 规则，放在属性值与分号之间，与值之间有空格，如果一个样式属性和值带有!important不会被其他样式覆盖，优先级最高，内联中不能添加

调错：

  1. unknown property name 属性名错误
  2. invalid property value 属性值写错了

四、基础选择器

作用：规范了页面中那些元素能够使用声明好的样式，目的就是为了匹配页面的元素
分类

  1. 通用选择器
    作用：匹配页面中的所有元素
    语法：*{样式声明}
    例如：*{margin:0;padding:0}
    注意：为了让代码可以适应各个浏览器，我们一般在开发前，把默认样式一次性清空或者统一，不让不同浏览器出现偏差，这个行为叫css reset

  2. 元素选择器,标签选择器
    作用：设置页面中所有这个元素的公用样式
    语法：div{样式声明} span{样式声明}

  3. ID选择器
    作用：匹配页面中对应的id的这个元素的样式
    语法：#id值{样式声明} <any id="id值"></any>

  4. 类选择器
    作用：定义多个元素的相同的样式，允许被元素的class属性进行引用的选择器
    语法：.类名{样式声明}
    类名规范：
    ①声明的时候，必须写点不能省略
    ②class调用的时候不能有点
    ③不能用数字开头
    ④类名中只能有两种符号- _
    例如：

    <style>
    .redcolor{ 
      color:red;
    }
    </style>
    <div  class="redcolor"></div>

    类选择器的其他引用方式
    a. 多类选择器
    语法：<any class="类名1 2 3 ">

    b. 分类选择器的定义方式
    语法：
    元素选择器.类名{样式声明} 匹配具体某一个元素，同时这个元素引用这个类名
    .类名1 .类名2{样式声明} 同一个标签，引用这两个类名，就可以引用这个样式
    作用：精确查找目标元素，提升选择器的权值

  5. 群组选择器
    特点：将多个选择器放在一起进行声明
    例如：

    div,p,#top,div.main{
      background-color:yellow;
    }

  6. 后代选择器
    特点：通过元素的层级关系来匹配元素，无限的层级关系都可以称之为后代。包含的选择器都会应用该样式
    语法：选择器1 选择器2{}
    例如：div p span {color:red}
    后代的执行效率比子代低那么一点点 不过可以忽略

  7. 子代选择器
    特点：通过元素的层级关系来匹配元素,只具备一层层级关系的元素成为”子代”
    例如：div>p>span {color:red}

  8. 伪类选择器
    作用：匹配某元素不同的状态
    ①链接伪类
    :link 定义超链接未被访问时的样式效果
    :visited 定义超链接访问后的样式效果
    ②动态伪类
    :hover 定义鼠标悬停在元素上时的效果
    :active 定义元素被激活的瞬间的样式效果(被点击的一瞬间)
    :focus 定义元素获取焦点时的样式效果
    :checked 定义元素被选中
    :disabled 定义元素被禁用

  9. 选择器的权值计算
    权值表示当前元素的重要程度，权值越大，优先级越高，越先显示；权值相同就近原则
    第一等：代表内联样式，如: style=""，权值为1000。
    第二等：代表ID选择器，如：#content，权值为0100。
    第三等：代表类，伪类和属性选择器，如.content，权值为0010。
    第四等：代表类型选择器和伪元素选择器，如div p，权值为0001。
    通配符、子选择器、相邻选择器等的。如*、>、+,权值为0000。
    继承的样式没有权值,!important权值>1000。
    群组选择器不想加各算各的，内联中不能添加!important

五、尺寸与边框

单位

  1. 尺寸单位
    ①px(像素)
    ②pt(磅)
    ③in(英寸)
    ④cm(厘米)
    ⑤mm(毫米)
    1pt=1/72in 1in=2.54cm
    ⑥相对单位
    % 相对于父元素数值的百分比
    em 父元素文字大小的整倍数
    rem 相对于html元素文字大小的整倍数

  2. 颜色单位
    ①RGB 红(R)、绿(G)、蓝(B) rgb(r,g,b) rgba(r,g,b,a)
    a指的是alpha透明度，取值0~1
    ②Hex #rrggbb
    rgb(0,102,153)=>#006699 16进制转换

尺寸

  1. 尺寸属性
    width 宽度，px为单位，%为父元素宽度百分比
    height 高度，px为单位，%为父元素高度百分比
    max-width/height 最大宽度/高度
    min-width/height 最小宽度/高度
    注意：
    ①父元素如果没有设置高度%失效
    ②max-width:100% 代码元素可以缩放，但是最大宽度不能超过原始尺寸
  2. 页面中可以设置尺寸的元素
    块级元素 行内元素 行内块
    设置宽高有效 除img之外都不允许修改尺寸 默认自带宽高(设置有效)
    没设置宽，默认宽是父元素100%；没设置高，默认高靠内容撑开 实际宽高靠内容撑开 默认自带宽高，不同浏览器对input的默认样式设置区别较大

总结：所有行内块都可以设置宽高，行内块自带宽高也可以设置宽高，多数行内块设置宽高无效；属性中自带宽高的标签，都可以设置样式的宽高(img,table)

  1. 溢出的处理
    原因：当内容较大，元素区域较小的时候会发生溢出（默认是纵向溢出）
    属性：overflow
    取值：
    ①visible 溢出部分显示（默认缺省值）
    ②hidden 溢出部分隐藏
    ③scroll 添加滚动条，不管是否溢出
    ④auto 根据是否溢出，自动添加滚动条
    overflow-x:scroll; 单独设置x轴滚动条
    overflow-y:scroll; 单独设置y轴滚动条

如何横向溢出？
在宽度比较小的容器里面设置一个宽度较大的子元素

边框border
最简：border:width style color

  1. 边框样式border-style
    ①none 无边框
    ②solid 实线
    ③dotted 虚线(点状)
    ④dashed 虚线(线状)
    ⑤double 定义两个边框，两个边框的宽度和 border-width 的值相同
    ⑥groove 定义3D沟槽边框，效果取决于边框的颜色值
    ⑦ridge 定义3D脊边框，效果取决于边框的颜色值
    ⑧inset 定义一个3D的嵌入边框。效果取决于边框的颜色值
    ⑨outset 定义一个3D突出边框。 效果取决于边框的颜色值
  2. 边框宽度border-width(设置长度值即可)
  3. 边框颜色border-color(颜色名称/RGB/Hex/透明transparent)
  4. 边框单边border-方向
    border-方向:width style color
    方向：top/right/bottom/left
  5. 边框倒角border-radius(以px为单位的数字或者%，50%是一个圆)
    单角设置：border-top/bottom-left/right-radius
    取消倒角：border-radius:0
  6. 边框阴影box-shadow
    ①h-shadow 水平方向上的阴影偏移
    ②v-shadow 垂直方向上的阴影偏移
    ③blur 阴影模糊面积
    ④spread 阴影大小
    ⑤color 阴影颜色
    ⑥inset 阴影重叠部分显示
    最简：box-shadow:h-shadow v-shadow;
  7. 轮廓outline
    边框的边框，绘制边框外面的线条，不占据元素空间
    outline:width style color;
    一般清除轮廓 outline:0;

六、框模型–盒子模型

元素在页面上实际占地空间的计算方式：
①默认实际宽度 = 左外边距+左边框+左内边距+内容区域宽度+右边框+右内边距+右外边距
②默认实际高度 = 上外边距+上边框+上内边距+内容区域高度+下边框+下内边距+下外边距

  1. 外边距margin(位于边框以外的距离(元素之间的间隔))
    margin 设置四个方向的外边距
    margin-top/right/bottom/left 设置某一侧的外边距
    取值：
    ①px为单位的数值
    ②以%为单位的数值 以父元素的高度或者高度的占比来作为外边距
    ③auto 让浏览器自己去计算元素的水平外边距
    让元素水平居中？
    可以使用auto，但是要满足两个条件
    1)必须为元素设置宽度
    2)必须是块级元素
    简洁语法
    ① margin:value; 四个方向都设置;
    ② margin:v1 v2; 上下 左右;
    ③ margin:v1 v2 v3; 上 左右 下;
    ④ margin:v1 v2 v3 v4; 上 右 下 左;

  2. 外边距的特殊效果
    1). 具有默认外边距的元素
    body h1-h6 p ol ul dl dd pre
    2). 外边距合并
    两个垂直外边距相遇时以大的为准
    解决办法：
    ①布局设计的时候进行规避
    ②只写一个垂直外边距，一下把数值写满
    3). 外边距的溢出
    在某些特殊情况下，给子元素设置上外边距时，会作用到父元素的上面，
    特殊：
    ①父元素没有上边框
    ②子元素内容区域的上沿与父元素内容区域的上沿重合(为第一个子元素设置上外边距)
    解决办法：
    ① 给父元素增加上边框 弊端：会影响父元素的高度
    ② 使用父元素的上内边距来取代子元素的上外边距 弊端：会影响父元素的高度
    ③ 添加一个空的table子元素，其余不行

    为行内元素以及行内块元素设置垂直外边距时
    1)行内元素对垂直外边距无效,横向有效
    2)行内块元素设置垂直外边距时，会导致整行元素都跟着变

  3. 内边距padding(边框到内容区域的距离）
    padding 设置四个方向的内边距
    padding-top/right/bottom/left 设置某一侧的内边距
    取值：
    ①px为单位的数值
    ②以%为单位的数值 以父元素的宽度或者宽度的占比来作为内边距
    注意
    改变内边距时，实际宽高不变；元素宽高，占地面积变大
    1)必须为元素设置宽度
    2)必须是块级元素
    简洁语法
    ① padding:value; 四个方向都设置;
    ② padding:v1 v2; 上下 左右;
    ③ padding:v1 v2 v3; 上 左右 下;
    ④ padding:v1 v2 v3 v4; 上 右 下 左;

  4. box-sizing
    作用：重新指定框模型的计算方式
    取值：

    div{
      width:200px;
      height:200px;
      margin:1px;
      padding:1px;
      border:solid blue 5px;
    }

    ①content-box 默认值，实际占地宽高=外边距+边框+内边距+width

    ②border-box 给定的width中，会包含padding以及border,也就是说实际占地宽高=外边距+width

七、背景

  1. 背景色
    属性：background-color
    取值：合法的颜色值
    注意：背景颜色是从边框的位置处开始绘制的

  2. 背景图片
    属性：background-image
    取值：url(背景图地址)

  3. 背景图片重复
    属性：background-repeat
    取值：
    ①repeat 默认值,重复
    ②repeat-x x轴平铺
    ③repeat-y y轴平铺
    ④no-repeat 不平铺(从左上角开始显示)

  4. 背景图片的尺寸
    属性：background-size
    取值：
    ①一个以px为单位的数字或者%，控制x轴和y轴
    ①两个以px为单位的数字或者%，分别控制x轴和y轴
    ③cover 覆盖 将背景图等比放大，直到覆盖到元素的所有区域为止，图片不完整
    ④contain 包含 会将背景图像等比放大，只要背景图的一个边缘碰到元素的边缘为止，图片完整

  5. 背景图片的固定
    属性：background-attachment
    取值：
    ①scrolll 默认值 背景图随着页面滚动条滚动
    ②fixed 固定 将背景图固定在浏览器的可视区域内，不会随着滚动条而发生位置的改变，只会在自己元素中显示

    取值为fixed时
    background-position就是相对于body定位，而不是相对于本身所在元素

  6. 背景图片的位置
    属性：background-position
    取值：
    ①一个以px为单位的数字或者%，控制x轴和y轴
    ①两个以px为单位的数字或者%，分别控制x轴和y轴
    ③关键字x轴 left/center/right y轴top/center/bottom

  7. 背景的简写方式
    属性：background
    取值：color img repeat attactment position;
    注意：
    ①简写中没有background-size
    ②最简background:color/img
    ③重写样式时使用简写会覆盖其他值

八、渐变

定义：多种颜色平缓变化的一种显示效果
色标：表示一种颜色及其出现的位置 渐变至少由两个色标组成
属性：background-image
渐变的分类

  1. 线性渐变：以直线的方向来填充渐变色
    取值：linear-gradient(方向,色标1,色标2);
    方向：
    1)关键字
    ①to top 从下向上填充渐变色
    ②to right
    ③to bottom
    ④to left
    2)角度值：
    ①0deg to top
    ②90deg to right
    ③180deg to bottom
    ④270deg to left
    色标：至少出现两个
    取值：颜色 以及位置的组合，中间用空格隔开
    例如：
    ①red 0% 开始的位置处红色
    ②blue 50% 到填充方向一半的位置处，变为蓝色
    ③green 200px 填充方向的200px位置处变为绿色

    .gradient {
    background-image:
        linear-gradient(
            to bottom,
            rgba(255, 255, 255, 0.1) 0%,
            rgba(0, 0, 0, 0.1) 40%,
            rgba(0, 0, 0, 0.2) 98%,
            #FFFFFF 100%
        );
    }

  2. 径向渐变：以圆形的方式实现渐变效果
    取值：radial-gradient(半径 at 圆心x 圆心y,色标);
    圆心取值：
    1)关键字
    ①x:left/center/right
    ②y:top/center/bottom
    2)以px为单位的数字
    3)%
    例如：

    .gradient {
    background-image:
        radial-gradient(
            2 at top right,
            yellow,
            #f06d06
        );
    }

  3. 重复渐变：将线性渐变和径向渐变重复实现多次
    1)线性重复：repeating-linear-gradient
    2)径向重复：repeating-radial-gradient
    例如：

    .repeat {
    background-image:
        repeating-linear-gradient(
            45deg,
            yellow,
            yellow 10px,
            red 10px,
            red 20px
        );
    }
    .repeat {
    background-image:
        repeating-radial-gradient(
        circle at 0 0, 
        #eee,
        #ccc 50px
        );
    }

  4. 浏览器的兼容性问题
    通过追加前缀让浏览器支持渐变
    不同浏览器的前缀：
    Firefox：-moz-
    chrome&safari：-webkit-

十二、定位

概念：改变元素在页面中的位置
分类：
1)普通流定位 2)浮动定位 3)相对定位 4)绝对定位 5)固定定位

  1. 普通流定位
    默认文档流定位
    1.每个元素在页面上都有自己的空间
    2.每个元素都是父元素的左上角开始显示
    3.块级元素按照从上往下的方式逐个排列,每个元素独立成行
    4.行内,行内块元素是多个元素在一行中显示,从左往右排列,一行放不下,换行
  2. 浮动定位float
    块级元素横向显示
    取值：
    ①left 左浮动
    让元素脱离文档流后,在当前行向父元素的左边对齐
    ②right 右浮动
    让元素脱离文档流后,在当前行向父元素的右边对齐
    ③none 不浮动,还在文档流里
    特点：
    1)元素一旦浮动,脱离文档(不占页面空间,后面元素上前补位)
2)浮动元素,最终会停靠在父元素的左边/右边,或者其他已浮动元素元素边缘
3)父元素横向显示不下所有浮动元素时,显示不下的元素,自动换行
4)浮动解决多个块级元素在同一行显示的问题
  3. 浮动引发的特殊情况
    ①浮动元素的占位问题
    当父元素显示不下所有浮动元素时,最后显示不下会换行,但是,已浮动元素会根据自己的浮动方向占据位置,导致被挤下去的元素,需要躲开这个占位显示
    ②元素一旦浮动,如果元素未定义宽度,那么元素的宽度靠内容撑开
    ③元素一旦浮动,变为块级元素(设置宽高有效,上下外边距有效)
    ④文本,行内元素,行内块,是不会被浮动元素压在下面,而且巧妙的避开浮动元素显示
  4. 清除浮动 clear
    清除之前浮动元素带来的影响(上前补位)
    取值：
    ①left 清除左浮动的影响
    ②right 清除右浮动的影响
    ③both 清除左右浮动的影响
  5. 高度坍塌
    父元素不写高度,内部子元素都浮动,父元素就找不到高了
    解决方案：
    ①父元素写高度,弊端,很多时候,高度确定不了
    ②父元素也浮动,弊端,会影响父元素的兄弟元素
    ③overflow:auto/hidden
    ④在父元素中追加一个空的块级子元素,为其定义clear:both;
  6. 脱离文档流
    ①元素不占据页面空间,后面元素上前补位
    ②元素一旦脱离文档流,元素就变为块级元素
    ③元素脱离文档流,元素没有设置宽度,元素的宽度靠内容撑开
  7. 显示方式 display
    取值：
    ①none 隐藏
    ②block 块级
    ③inline 行内
    ④inline-block 行内块
    ⑤table 表格
  8. 显示效果 visibility
    取值：
    ①默认值 visible 可见的
    ②hidden 隐藏的
  9. 透明度 opacity
    取值： 0 全透明 1 不透明
  10. 垂直对齐方式 vertical-align
    场景：
    ①用在table,控制table中的内容的垂直对齐方式
    vertical-align:top/middle/bottom
    ②图片,给图片设置垂直对齐,图片本身不动,影响的是图片前后的文本位置.
    vertical-align:top/middle/bottom/baseline(默认)
    项目中,经常为图片设置vertical-align,让文本不以基线对齐
  11. 光标的设置 cursor
    取值：
    ①default 箭头
    ②pointer 小手
    ③crosshair +
    ④help 帮助
    ⑤text 文本
    ⑥wait 等待
  12. 列表的样式 list-style
    场景：
    ①设置列表标识项
    list-style-type:disc(实心圆点)/circle(空心圆点)/square(方形点)/none;
    ②标识项设置为图片
    list-style-image:url(true.png);
    ③列表项的位置
    list-style-position:inside/outside(默认值);
    ④简写方式
    list-style:type image position;
    ⑤去掉样式
    项目中,使用最频繁的方式 list-style:none;
    ⑥无序列表在项目中地位和使用方式
    ul多用于布局
  13. 定位 position
    取值：
    ①static 静态,默认值,文档流
    ②relative 相对定位
    ③absolute 绝对定位
    ④fixed 固定定位
    注意：
    ①一个元素被position修饰,并且取值为relative/ absolute/ fixed
    此元素被称为已定位元素
    ②一个元素变成已定位元素之后,
    解锁4个偏移属性
    top: 离顶部多远 + ↓ - ↑
    right:
    bottom:
    left: 离左边多远 + → - ←
    1)相对定位 position:relative;
    相对定位不脱离文档流,对周围元素的位置没有任何影响
如果相对定位不写偏移属性,效果与没写定位一样
相对自已原来位置做偏移
使用场合
①类似于margin,做自身位置的微调
②作为绝对定位的祖先级已定位属性
    2)绝对定位 position:absolute;
    绝对定位脱离文档流
**场景：**
①没有已定位的祖先级元素,相对body左上角偏移 
②相对于,最近的,已定位的,祖先级元素做偏移
**注意:**通常把祖先级变成已定位元素的样式,添加定位`position:relative;`
    3)固定定位 position:fixed;
    将元素固定在页面上某个位置,不会随着滚动条滚动,变化位置
一直固定在可视区域
    4)堆叠顺序 z-index
    无单位正整数
**注意:**
①定位的堆叠和浮动的脱离文档流,不是一个体系
②默认,后声明的定位元素,堆叠顺序高,但是不会超过1
③z-index只对已定位元素生效
④堆叠顺序对父子级元素无效,儿子永远在父亲的上面

一、复杂选择器

  1. 复杂选择器
    兄弟元素:具备相同父元素的平级元素,称之为兄弟元素
    兄弟选择器,只能通过哥哥找弟弟,不能往前找
    ①相邻兄弟选择器
    选择器1+选择器2{}获取紧紧挨着某元素后面的兄弟元素
    ②通用兄弟选择器
    选择器1~选择器2{}获取某元素后面所有符合要求的兄弟元素
    使用场景,多用于淘汰第一个元素,其它兄弟都使用的样式
  2. 属性选择器
    [attr]{} 匹配定义了attr这个属性的元素
    [attr1] [attr2]{} 匹配同时定义了attr1和attr2属性的元素
    [attr=value] 匹配定义了attr同时值为value的元素
    [attr1=value1] [attr2=value2]{}
    elem[attr]{} 匹配定义了attr的elem元素
    elem[attr=value]{} 模糊属性值
    [attr^=value]{} 匹配attr值的开头是value的元素
    [attr$=value]{} 匹配attr值的结尾是value的元素
    [attr*=value]{} 匹配attr值里有value的元素
    [attr~=value]{} 匹配attr值有value这个单词(前后有空格)的元素
  3. 伪类选择器
    ①目标伪类:target{}
    让被激活的锚点,应用的样式
    ②结构伪类
    selector:first-child 匹配属于selector父元素的第一个子元素,
     并且这个元素还得符合selector
(找到是selector的大哥,这个大哥也得符合selector)
    selector:last-child 匹配属于selector父元素的最后一个子元素,
     并且这个元素还得符合selector
(找到是selector的小弟弟,这个小弟弟也得符合selector)
    :nth-child(n) 匹配属于父元素的第n个子元素 n是1开始
    ③:empty 匹配内部没有任何元素的标签
    注意:所谓没有元素,包括不能有空格,回车,标签,文本
    ④:only-child 属于其父元素的唯一子元素
    可以有其它的文本,空格,回车,没有其它标签就算独生子女
    ⑤否定伪类
    :not(selector){}
    符合selector都不要
  4. 伪元素选择器
    ::first-letter 或者 :first-letter
    匹配元素的首字符
    ::first-line 或者 :first-line
    匹配元素的首行
    ::selection只有双冒号
    匹配用户选中的内容
    只能设置字体颜色和背景颜色
  5. 内容生成
    ::before 或者 :before
    在元素内容区域内部的最前端,添加一个假的元素.
    这个假元素,有属性content,设置该元素中显示的内容
    content中只能放文字和图片url()
    在这个假元素中,我们可以使用display设置显示规则.
    其它样式也会生效
    ::after 或者 :after
    在元素内容区域内部的最后端,添加一个假的元素.
    内容生成,可以解决的问题
    ①.解决外边距溢出 
    父元素:before{
    content:"";
    display:table;
    }
    ②.解决高度坍塌 
    #parent:after{
    content:"";
    display:block;
    clear:both;
    }

    二.弹性布局

  6. 什么是弹性布局
    是一种布局方式,主要解决某个元素中,子元素的布局方式,非常的灵活
  7. 弹性布局的相关概念
    ①容器
    要发生弹性布局的子元素,他们的父元素称为容器
    设置了display:flex的那个元素,就是容器
    ②项目
    要发生弹性布局的子元素们,叫做项目
    ③主轴
    项目们的排列方向,就是主轴
    项目们横向排列,主轴就是x轴
    项目们纵向排列,主轴就是y轴
    项目们在主轴上的排列顺序,称为主轴的起点和终点
    ④交叉轴
    与主轴方向,垂直的方向,称为交叉轴
    项目们在交叉轴的对齐方式,称为交叉轴的起点和终点
  8. 语法
    将元素设置成弹性容器后,他的所有子元素将变成弹性项目
    都允许按照弹性布局的方式排列
    display:
    ①.`flex` 将块级元素设置为容器
②.`inline-flex` 将行内元素设置为容器
    特点：
    ①.容器的text-align/vertical-align失效
    ②.项目的float/clear 失效
    总结,项目中的水平对齐和垂直对齐,只能够靠主轴和交叉轴控制
  9. 容器的属性
    ①主轴方向
    flex-direction:
    取值
    row 默认值,主轴是x轴,主轴起点是左侧
    row-reverse主轴是x轴,主轴起点是右侧
    column 主轴是y轴,主轴起点是上
    column-reverse 主轴是y轴,主轴起点是下
    ②设置项目的换行
    容器空间不足是,项目是否换行
    flex-wrap:nowrap 默认值,不换行
    `wrap`    换行
`wrap-reverse` 换行并反转
    ③flex-flow
    flex-direction和flex-wrap的缩写
    flex-flow: direction wrap;
    ④项目在主轴上的对齐方式
    justify-content:flex-start 默认值,主轴起点
    `center`    主轴中心
`flex-end`  主轴终点
`space-around` 每个间距大小相同,两端有距离
`space-between` 两端贴边,每个间距大小相同
    ⑤交叉轴的对齐方式
    align-items:flex-start 默认,交叉轴起点对齐
    `center` 交叉轴中心
`flex-end`  交叉轴终点
`stretch`  前提,项目不写高,沾满交叉轴
  10. 项目的属性,设置在某一个项目上,不影响其他项目
    ①项目的排列顺序
    order:无单位的整数
    数字越小,项目离主轴起点越近
    ②flex-grow
    主轴方向,有剩余空间,项目变大
    flex-grow:默认值为0,不变大
    取值数字越大,变大的越快
    ③flex-shrink
    主轴的方向,空间不够,项目缩小
    默认值是 1
    取值数字越大,变小的越快
    ④单个项目交叉轴对齐方式
    align-self:
    `flex-start` 默认,交叉轴起点对齐
`center` 交叉轴中心
`flex-end`  交叉轴终点
`stretch`  前提,项目不写高,沾满交叉轴
`auto`    继承容器的`align-items`的值

  11. 什么是转换

    3d  1个

  13. 2D转换
    ①位移
    transform:translate()
    取值:`translate(v1)`和`translateX(v1)`,都是x轴位移
        `translateY(v1)`  y轴的位移
        `translate(v1,v2)` v1是x轴,v2是y轴
            x:+ →,  -  ←
            y:+ ↓ ,  - ↑
    ②缩放
    transform:scale(v1) x轴和y轴同时缩放
    `scale(v1,v2)`   x轴和y轴分别设置缩放
`scaleX(v1)`  x轴缩放
`scaleY(v1)`  y轴缩放
`v>1`  放大   
`0<v<1` 缩小
`-1<v<0` 反转并缩小
`v<-1`   反转并放大
    ③旋转
    transform:rotate(ndeg) n:+ 顺时针, - 逆时针
    注意:1.转换原点会改变转换效果,默认元素中心
    `transform-origin:right center;`
取值:①.%
     ②.关键字 x:`left/center/right`
              y:`top/center/bottom`
     ③.取值数量:2个值,原点在x轴y轴上
               3个值,添加了z轴
2.旋转是连同坐标轴一起旋转的,旋转之后,会影响位移方向
    ④倾斜
    transform:skew()
    1.skew(45deg)和skewX(45deg)
    y轴向着x轴倾斜45
    • 逆时针 - 顺时针
    2.skewY(45deg)
    x轴向着y轴倾斜45
    • 顺时针 - 逆时针
    3.skew(45deg,45deg)
  14. 3D转换
    3D转换都是模拟的
    ①透视距离
    模拟人的眼睛到3D转换的元素之间的距离,距离不同,效果不同
    perspective:距离 此属性,要定在3d转换元素的父元素上
    ②3D旋转
    transform:
    取值:1.rotateX(xdeg) 以x轴为中心,旋转元素角度
                    烤羊腿,老式爆米花机
2.`rotateY(ydeg)` 以y轴为中心,旋转元素角度
                旋转木马,旋转门,土耳其烤肉
3.`rotateZ(zdeg)` 以z轴为中心,旋转元素角度
                车轮,风车,印度飞饼,摩天轮
4.`rotate3D(x,y,z,ndeg)`
        x,y,x 取值为0,代表不参与旋转
                >0 代表参与旋转

    五.过渡

  15. 过渡:让css的属性值,在一段时间内,平缓的变化(css是两个值)
  16. 语法
    transition-property 指定使用过渡的属性
    取值,css的属性名称,多个属性名称之间,用空格分开
    `all`,所有支持过渡的属性,都参与此次过渡
    注意,transform必须执行过渡效果
    支持过渡的属性:
    ①.颜色属性
②.大多数取值为具体数字的属性
③.阴影
④.转换
⑤.`visibility`
  17. transition-duration 过渡效果持续的时长 s/ms
  18. 过渡时间曲线函数
    transition-timing-function:
    ease:默认值,慢速开始,中间加速,慢速结束
    linear:匀速
    ease-in:慢慢开始,一直加速
    ease-out:快速开始,一直减速
    ease-in-out:慢开,加速到中间,减速,慢关
    steps(8)
  19. 过渡的延迟 transition-delay:1s;
  20. 过渡代码编写的位置
    写在原来的样式中,过渡效果有去有回
    写在hover,过渡效果有去无回
  21. 过渡的简写方式
    transition:property duration timing-fucntion delay;
    最简方式 transition:duration;
  22. 总结:什么时候使用过渡
    ①css的属性值,只有两个状态(起始状态,终止状态)
    ②css的变化,只能用hover激活(终止状态必须写在hover里)

    六.动画

  23. 什么是动画
    多个过渡效果放到一起,就是动画
  24. 使用关键帧,来控制动画的每一个状态
    关键帧 1.动画执行的时间点
    2.在这个时间点上的样式
  25. 定义动画 
    @keyframes 动画名称{
    0%{样式}
    ...
    100%{样式}
    }
  26. 调用动画 
    /*1.使用动画名称*/
    animation-name:change1;
    /*2.动画执行的时间*/
    animation-duration:3s;
    /*3.动画执行的时间曲线函数*/
    animation-timing-function:linear;
    /*4.动画执行的延迟*/
    animation-delay:1s;
  27. 动画的其它属性
    ①动画播放次数
    animation-iteration-count:具体数字/infinite无限;
    ②动画的播放顺序
    animation-direction:
    normal 0%—>100%
    reverse 100%—>0%
    alternate 播放的奇数次 0%—>100%
    偶数次100%--->0%
    ③动画的简写方式
    animation:name duration timing-function delay count direction;
    最简方式
    animation:name duration;
    ④动画的填充状态
    animation-fill-mode:
    backwards 动画的延迟状态中,显示动画第一帧
    forwards 动画结束后,显示动画最后一帧
    both 开头结尾都填充
    none 不填充
    ⑤动画的播放状态
    animation-play-state:
    paused 暂停
    running 播放
  28. 动画的兼容性
    定义动画的时候,写css hack 
    @keyframes 动画名称{}
    @-webkit-keyframes 动画名称{}
    @-moz-keyframes 动画名称{}
    @-o-keyframes 动画名称{}
    @-ms-keyframes 动画名称{}
  29. 什么时候用动画,什么时候用过渡
    ①超过两个css的样式状态,必须用动画
    ②不使用hover触发,F5刷新,直接出效果,必须用动画
    ③如果只有两个状态,并且hover触发,动画和过渡都可以使用，推荐使用过渡
  30. animate.css的使用
    https://daneden.github.io/animate.css/
    提供了一个动画css库

    七.CSS优化

    css优化的目的
    1.减少服务器端的压力
    2.提升用户体验
    css优化的原则
    尽量的减少http的请求个数(缓存) 使用精灵图
    页面的顶部,引入css文件
    将css和js写在单独的文件中
    css代码优化
    1.合并样式(能重用,就重用,能群组,就不单独定义,能用简写,就不单写)
    2.代码压缩
    3.避免出现空的src和href

    八.样式重置

    不同浏览器对元素默认的样式有很大差异
  31. css reset
    比较霸道,不管你是不是需要html的原始样式,都清掉
    我们可以自已编写css reset,也可以使用第三方别人写好的
    先把所有样式都清除,让元素基本都没有
    但是需要基本的样式的时候,还得手动添加,浪费资源
  32. normalize.css
    是一种非常流行的css reset的解决方案
    重写样式比较柔和
    保存了很多默认样式,提升了css的兼容性,修复了很多浏览器的bug
    使用： 
    <link rel="stylesheet" href="http://cdn.staticfile.org/normalize/3.0.1/normalize.min.css" type="text/css" />
